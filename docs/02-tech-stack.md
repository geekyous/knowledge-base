## 📊 数据存储架构

### MySQL（业务数据）
```
用途：存储结构化业务数据
├── 用户数据：users, roles, permissions
├── 文档数据：documents, categories, tags
├── 内容数据：document_content, versions
└── 日志数据：operation_logs, access_logs
```

### Redis（缓存和会话）
```
用途：缓存热点数据和会话管理
├── 用户会话：user:session:{userId}
├── API 缓存：api:cache:{key}
├── 搜索缓存：search:cache:{key}
└── 分布式锁：locks:{resource}
```

### Elasticsearch（全文搜索）
```
用途：文档全文搜索和索引
├── 文档索引：documents-index
├── 用户索引：users-index
└── 日志索引：logs-index
```

### Qdrant（向量存储）
```
用途：文档向量存储和相似度搜索
├── 文档向量：documents_collection
└── 问答历史：conversations_collection
```

RestTemplate restTemplate = new RestTemplate();

## 🎯 技术选型理由

### 为什么选择 Vue 3？
- ✅ 组合式 API 更灵活
- ✅ TypeScript 支持优秀
- ✅ 性能优于 Vue 2
- ✅ 团队熟悉度高

### 为什么选择 Spring Boot？
- ✅ 企业级标准框架
- ✅ 生态成熟完善
- ✅ 团队 Java 背景强
- ✅ 安全性有保障

### 为什么分离 AI 服务？
- ✅ Python AI 生态丰富
- ✅ 独立部署和扩展
- ✅ 技术解耦
- ✅ 便于 AI 模型升级

### 为什么选择 MySQL？
- ✅ 企业级关系数据库
- ✅ 事务支持完善
- ✅ 团队使用经验
- ✅ 云服务支持好

### 为什么选择 Redis？
- ✅ 高性能缓存
- ✅ 丰富的数据结构
- ✅ 支持分布式场景
- ✅ 会话管理方便

## 📈 性能考量

### 前端性能
- 虚拟滚动（长列表）
- 懒加载组件
- 图片懒加载
- 代码分割

### 后端性能
- Redis 缓存热点数据
- 数据库连接池
- 异步处理非核心业务
- 接口限流

### AI 服务性能
- 向量缓存
- 批量处理
- 异步任务队列
- 结果缓存
