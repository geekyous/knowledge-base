# F05 - 用户个人中心 - 功能设计文档

## 📋 功能概述

用户个人中心为用户提供个人信息管理、统计数据展示、活动记录查看等功能，增强用户参与感和归属感。

## 🎯 用户故事

### 1. 个人信息管理
**作为** 系统用户  
**我想要** 查看和修改个人信息  
**以便** 保持账号信息准确性

**验收标准：**
- [ ] 查看个人基本信息
- [ ] 修改用户头像
- [ ] 修改邮箱和手机号
- [ ] 修改个人简介

### 2. 个人统计
**作为** 系统用户  
**我想要** 查看个人贡献统计  
**以便** 了解自己的知识贡献

**验收标准：**
- [ ] 显示发布文档数
- [ ] 显示贡献答案数
- [ ] 显示获得点赞数
- [ ] 显示关注者数量
- [ ] 显示排名信息

### 3. 活动记录
**作为** 系统用户  
**我想要** 查看个人活动历史  
**以便** 回顾自己的操作记录

**验收标准：**
- [ ] 显示时间轴活动记录
- [ ] 显示操作类型和描述
- [ ] 支持按时间筛选
- [ ] 显示操作结果

### 4. 个人设置
**作为** 系统用户  
**我想要** 管理个人设置  
**以便** 优化使用体验

**验收标准：**
- [ ] 修改密码
- [ ] 设置通知偏好
- [ ] 隐私设置
- [ ] 界面主题设置

## 🔐 功能需求

### 1. 个人信息头部

#### 功能描述
显示用户的基本信息和成就，提供视觉化的个人中心入口。

#### 展示信息
```
┌────────────────────────────────────────┐
│         [头像]                         │
│        张三                            │
│   产品经理 | 产品管理部                  │
│   [精英用户] [知识贡献者]             │
├────────────────────────────────────────┤
│  发布文档: 23 | 贡献答案: 156           │
│  获得点赞: 1.2k | 关注者: 89            │
└────────────────────────────────────────┘
```

#### 成就徽章
- **精英用户**：高质量内容贡献者
- **知识贡献者**：文档数>20或答案数>100
- **活跃用户**：登录天数>30天
- **新星用户**：注册<30天但活跃度高

### 2. 统计数据

#### 数据指标

| 指标 | 说明 | 数据来源 |
|------|------|----------|
| 发布文档数 | 用户创建的文档数量 | documents表 |
| 贡献答案数 | 用户回答的问题数量 | messages表 |
| 获得点赞数 | 收到的点赞总数 | likes表 |
| 关注者数 | 关注该用户的人数 | follows表 |
| 热度 | 在贡献者中的排名 | 计算得出 |

#### 可视化展示
- **数字卡片**：大字体显示核心数据
- **趋势图**：显示贡献趋势
- **排名表**：显示用户排名

### 3. 最近活动

#### 活动类型

| 类型 | 图标 | 示例 |

| 获得点赞 | 👍 | 你的答案获得了10个点赞 |

### 4. 个人设置

#### 设置分类
- **账户设置**：用户名、邮箱、密码
- **通知设置**：通知类型、通知频率
- **隐私设置**：资料公开范围、活动记录
- **界面设置**：主题选择、语言设置

## 🔗 接口定义

### 1. 获取个人信息

**接口地址：** `GET /api/v1/users/me`

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "zhangsan",
    "email": "zhangsan@example.com",
    "phone": "13800138000",
    "avatar": "https://cdn.example.com/avatars/zhangsan.jpg",
    "role": "EDITOR",
    "status": "ACTIVE",
    "department": "产品管理部",
    "position": "产品经理",
    "bio": "负责产品规划和设计",
    "createdAt": "2024-01-01T00:00:00Z",
    "lastLoginAt": "2024-06-01T09:30:00Z"
  }
}
```

### 2. 更新个人信息

**接口地址：** `PUT /api/v1/users/me`

**请求参数：**
```json
{
  "email": "newemail@example.com",
  "phone": "13900139000",
  "avatar": "https://cdn.example.com/avatars/new-avatar.jpg",
  "bio": "新个人简介"
}
```

### 3. 获取用户统计

**接口地址：** `GET /api/v1/users/me/statistics`

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "documentCount": 23,
    "answerCount": 156,
    "likeCount": 1200,
    "followerCount": 89,
    "ranking": 5
  }
}
```

### 4. 获取活动记录

**接口地址：** `GET /api/v1/users/me/activities`

**请求参数：**
```json
{
  "page": 1,
  "pageSize": 20,
  "type": "all"
}
```

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "items": [
      {
        "id": 1,
        "type": "document",
        "action": "create",
        "description": "创建了文档《产品发布流程》",
        "targetId": 123,
        "targetTitle": "产品发布流程",
        "createdAt": "2024-06-01T10:30:00Z"
      },
      {
        "id": 2,
        "type": "answer",
        "action": "create",
        "description": "回答了问题\"如何申请年假\"",
        "targetId": 456,
        "targetTitle": "如何申请年假？",
        "createdAt": "2024-06-01T09:15:00Z"
      }
    ],
    "total": 156
  }
}
```

### 5. 修改密码

**接口地址：** `PUT /api/v1/users/me/password`

**请求参数：**
```json
{
  "oldPassword": "oldpass123",
  "newPassword": "newpass456"
}
```
