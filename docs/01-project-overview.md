# 企业知识库问答系统 - 项目概述

## 📋 系统简介

企业知识库问答系统是一个集文档管理、智能搜索、AI 问答于一体的企业级知识管理平台。系统通过现代化的技术架构，为企业提供高效的知识存储、检索和问答服务。

## 🎯 项目目标

### 主要目标
- 提高企业内部知识共享效率
- 降低重复问题解答成本
- 保存和传承企业核心知识
- 提供智能化的知识服务

### 业务价值
- **效率提升** - 员工可快速找到所需信息，节省查找时间
- **知识沉淀** - 将隐性知识转化为显性文档，形成企业知识资产
- **智能问答** - AI 助手 7x24 小时在线，即时回答常见问题
- **数据分析** - 通过用户行为分析，了解知识需求和内容质量

## 🏗️ 系统架构

### 整体架构

系统采用前后端分离的微服务架构，主要分为三部分：

```
┌─────────────────────────────────────────────────────┐
│                    前端层                             │
│  Vue 3 单页应用，负责用户交互和界面展示               │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                   API 网关层                          │
│  Nginx 反向代理，处理路由、负载均衡、静态资源          │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                   业务服务层                          │
│  Spring Boot 主服务：用户管理、文档管理、权限控制      │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                   数据存储层                          │
│  MySQL（业务数据）+ Redis（缓存）+ Elasticsearch      │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                   AI 服务层                          │
│  Python FastAPI：RAG 检索、LLM 问答、向量化           │
└─────────────────────────────────────────────────────┘
```

### 技术分层

| 层级 | 技术 | 职责 |
|------|------|------|
| **前端展示层** | Vue 3 + TypeScript | 用户界面、交互逻辑 |
| **网关层** | Nginx | 路由分发、负载均衡 |
| **业务服务层** | Spring Boot | 业务逻辑、数据处理 |
| **数据访问层** | JPA/MyBatis | 数据持久化 |
| **缓存层** | Redis | 热点数据缓存 |
| **搜索引擎** | Elasticsearch | 全文搜索 |
| **AI 服务层** | FastAPI + LangChain | 智能问答 |
| **向量数据库** | Qdrant | 文档向量存储 |

## 🔄 业务流程

### 文档管理流程
```
文档创建 → 内容编辑 → 分类标签 → 权限设置 → 提交审核 → 发布 → 搜索索引 → AI 向量化
```

### 智能问答流程
```
用户提问 → 意图识别 → 相关文档检索 → 向量相似度匹配 → 上下文构建 → LLM 生成答案 → 结果返回
```

### 用户访问流程
```
用户登录 → 权限验证 → 功能访问 → 操作记录 → 数据统计
```
