# F01 - 用户认证与权限管理 - 功能设计文档

## 📋 功能概述

用户认证与权限管理模块是企业知识库问答系统的基础功能，负责用户身份验证、授权和权限控制，确保系统安全性和数据隔离。

## 🎯 用户故事

### 1. 用户登录
**作为** 系统用户  
**我想要** 通过用户名和密码登录系统  
**以便** 访问知识库内容和系统功能

**验收标准：**
- [ ] 支持用户名/密码登录
- [ ] 登录失败时显示错误提示
- [ ] 登录成功后获取JWT Token
- [ ] Token存储在本地并自动附加到请求头
- [ ] Token过期时自动跳转登录页

### 2. 用户注册
**作为** 新用户  
**我想要** 注册系统账号  
**以便** 使用知识库功能

**验收标准：**
- [ ] 支持用户名、邮箱、密码注册
- [ ] 用户名唯一性校验
- [ ] 邮箱格式验证
- [ ] 密码强度要求（至少8位，包含字母和数字）
- [ ] 注册成功后自动登录

### 3. 权限控制
**作为** 管理员  
**我想要** 控制不同用户的访问权限  
**以便** 保护系统安全和敏感数据

**验收标准：**
- [ ] 三种角色：USER、EDITOR、ADMIN
- [ ] 基于角色的访问控制（RBAC）
- [ ] API接口权限验证
- [ ] 前端路由权限守卫
- [ ] 数据权限隔离

### 4. 用户信息管理
**作为** 已登录用户  
**我想要** 查看和修改个人信息  
**以便** 保持账号信息的准确性

**验收标准：**
- [ ] 查看个人资料
- [ ] 修改头像、邮箱等信息
- [ ] 修改密码（需要验证旧密码）
- [ ] 查看个人统计数据

## 🔐 功能需求

### 1. 登录功能

#### 功能描述
用户提供用户名和密码，系统验证身份后返回JWT Token，用于后续API请求的身份认证。

#### 业务流程
```
用户输入凭证 → 验证用户名密码 → 生成JWT Token → 返回用户信息和Token → 存储Token到本地 → 跳转首页
```

#### 输入说明
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 3-20个字符 |
| password | string | 是 | 6-32个字符 |

#### 输出说明
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "admin",
      "email": "admin@example.com",
      "role": "ADMIN"
    }
  }
}
```

#### 异常处理
- **400 Bad Request**: 参数验证失败
- **401 Unauthorized**: 用户名或密码错误
- **423 Locked**: 账号已被锁定

### 2. 注册功能

#### 功能描述
新用户通过注册页面创建账号，系统创建用户记录并初始化用户角色。

#### 业务流程
```
填写注册信息 → 验证输入格式 → 检查用户名唯一性 → 密码加密存储 → 创建用户记录 → 自动登录 → 跳转首页
```

#### 输入说明
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 3-20个字符，字母数字下划线 |
| email | string | 是 | 有效的邮箱格式 |
| password | string | 是 | 8-32个字符，包含字母和数字 |

#### 验证规则
```javascript
// 用户名验证
username: {
  pattern: /^[a-zA-Z0-9_]{3,20}$/,
  message: '用户名只能包含字母、数字和下划线，长度3-20个字符'
}

// 密码强度
password: {
  pattern: /^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d@$!%*#?&]{8,}$/,
  message: '密码至少8位，包含字母和数字'
}

// 邮箱格式
email: {
  pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
  message: '请输入有效的邮箱地址'
}
```

### 3. 权限控制

#### 角色定义

| 角色 | 权限范围 | 描述 |
|------|----------|------|
| **USER** | 基础权限 | 查看公开文档、搜索、提问、个人中心 |
| **EDITOR** | 编辑权限 | USER权限 + 创建/编辑文档、审核文档 |
| **ADMIN** | 管理权限 | EDITOR权限 + 用户管理、系统配置、数据统计 |

#### 权限矩阵

| 功能模块 | USER | EDITOR | ADMIN |
|----------|------|--------|-------|
| 查看公开文档 | ✅ | ✅ | ✅ |
| 搜索功能 | ✅ | ✅ | ✅ |
| AI问答 | ✅ | ✅ | ✅ |
| 个人中心 | ✅ | ✅ | ✅ |
| 创建文档 | ❌ | ✅ | ✅ |
| 编辑文档 | ❌ | ✅ | ✅ |
| 删除文档 | ❌ | ✅ | ✅ |
| 审核文档 | ❌ | ❌ | ✅ |
| 用户管理 | ❌ | ❌ | ✅ |
| 系统设置 | ❌ | ❌ | ✅ |
| 数据统计 | ❌ | ❌ | ✅ |

### 4. Token管理

#### JWT Token结构
```json
{
  "sub": "1",
  "username": "admin",
  "role": "ADMIN",
  "iat": 1717182345,
  "exp": 1717268745
}
```

#### Token刷新机制
- Token有效期：24小时（可配置）
- 自动刷新：Token过期前30分钟
- 刷新接口：`POST /api/v1/auth/refresh`

### 5. 安全机制

#### 密码安全
- 使用BCrypt加密存储
- 加密强度：10轮加盐哈希
- 不在日志中记录密码

#### 账号安全
- 登录失败5次自动锁定30分钟
- 支持管理员解锁
- 密码修改需要验证旧密码

#### Token安全
- 使用HTTPS传输
- Token存储在HttpOnly Cookie（可选）
- 支持Token黑名单（登出时）

## 🎨 UI设计规范

### 1. 登录页面

#### 页面布局
- 居中卡片式设计
- 宽度：400px
- 圆角：12px
- 背景：渐变色背景

#### 表单元素
- **输入框**：左侧图标、圆角边框、聚焦时蓝色边框
- **按钮**：主按钮使用主题色、宽度100%、圆角8px
- **链接**：忘记密码、注册账号

#### 交互反馈
- 输入框聚焦：阴影效果
- 按钮悬停：颜色加深
- 登录中：显示加载动画、按钮禁用
- 错误提示：红色文字、表单下方显示

### 2. 权限提示

#### 无权限访问
当用户访问无权限的资源时：
- 显示403错误页面
- 提示"您没有权限访问此页面"
- 提供"返回首页"按钮

#### 登录提示
当未登录用户访问需要登录的页面时：
- 显示401错误页面
- 提示"请先登录"
- 提供"去登录"按钮

## 📊 数据模型

### 用户表（users）

| 字段 | 类型 | 长度 | 约束 | 说明 |
|------|------|------|------|------|
| id | BIGINT | - | PK | 主键 |
| username | VARCHAR | 50 | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR | 255 | NOT NULL | 密码（BCrypt加密） |

## 🔗 接口定义

### 1. 用户登录

**接口地址：** `POST /api/v1/auth/login`

**请求参数：**
```json
{
  "username": "admin",
  "password": "password123"
}
```

**响应示例：**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "admin",
      "email": "admin@example.com",
      "role": "ADMIN",
      "avatar": "https://example.com/avatar.jpg"
    }
  },
  "timestamp": 1717182345678
}
```

### 2. 用户注册

**接口地址：** `POST /api/v1/auth/register`

**请求参数：**
```json
{
  "username": "newuser",
  "email": "user@example.com",
  "password": "Password123"
}
```

**响应示例：**
```json
{
  "code": 201,
  "message": "注册成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 2,
      "username": "newuser",
      "email": "user@example.com",
      "role": "USER"
    }
  }
}
```

### 3. 获取当前用户信息

**接口地址：** `GET /api/v1/users/me`

**请求头：**
```
Authorization: Bearer {token}
```

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "admin",
    "email": "admin@example.com",
    "role": "ADMIN",
    "avatar": "https://example.com/avatar.jpg",
    "lastLoginAt": "2024-06-01T10:30:00Z",
    "createdAt": "2024-01-01T00:00:00Z"
  }
}
```

### 4. 更新用户信息

**接口地址：** `PUT /api/v1/users/me`

**请求参数：**
```json
{
  "email": "newemail@example.com",
  "avatar": "https://example.com/new-avatar.jpg"
}
```

### 5. 修改密码

**接口地址：** `PUT /api/v1/users/me/password`

**请求参数：**
```json
{
  "oldPassword": "oldpass123",
  "newPassword": "newpass456"
}
```

### 6. 刷新Token

**接口地址：** `POST /api/v1/auth/refresh`

**请求头：**
```
Authorization: Bearer {token}
```

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

### 7. 登出

**接口地址：** `POST /api/v1/auth/logout`

**响应示例：**
```json
{
  "code": 200,
  "message": "登出成功"
}
```

## ⚠️ 边界情况和异常处理

### 1. 登录异常

| 异常情况 | HTTP状态码 | 错误信息 | 处理方式 |
|----------|-------------|----------|----------|
| 用户名不存在 | 401 | 用户名或密码错误 | 统一提示，防止枚举攻击 |
| 密码错误 | 401 | 用户名或密码错误 | 记录失败次数 |
| 账号已锁定 | 423 | 账号已锁定，请联系管理员 | 提示联系管理员 |
| Token过期 | 401 | 登录已过期，请重新登录 | 跳转登录页 |
| Token无效 | 401 | 无效的认证信息 | 跳转登录页 |

### 2. 注册异常

| 异常情况 | HTTP状态码 | 错误信息 | 处理方式 |
|----------|-------------|----------|----------|
| 用户名已存在 | 409 | 用户名已被使用 | 提示更换用户名 |
| 邮箱已注册 | 409 | 邮箱已被注册 | 提示更换邮箱或找回密码 |
| 密码强度不足 | 400 | 密码强度不符合要求 | 显示具体要求 |

### 3. 权限异常

| 异常情况 | HTTP状态码 | 错误信息 | 处理方式 |
|----------|-------------|----------|----------|
| 未登录 | 401 | 请先登录 | 跳转登录页 |
| 权限不足 | 403 | 您没有权限执行此操作 | 显示权限不足页面 |
| 资源不存在 | 404 | 请求的资源不存在 | 显示404页面 |
