# F06 - 管理后台 - 功能设计文档

## 📋 功能概述

管理后台为系统管理员提供全面的管理功能，包括数据统计、用户管理、内容审核、系统配置等核心管理能力。

## 🎯 用户故事

### 1. 数据统计
**作为** 系统管理员  
**我想要** 查看系统使用统计  
**以便** 了解系统运行情况

**验收标准：**
- [ ] 显示核心指标数据
- [ ] 显示使用趋势图表
- [ ] 支持时间范围筛选
- [ ] 数据实时更新

### 2. 用户管理
**作为** 系统管理员  
**我想要** 管理用户账号  
**以便** 控制用户访问权限

**验收标准：**
- [ ] 查看用户列表
- [ ] 创建/编辑/删除用户
- [ ] 修改用户角色
- [ ] 启用/禁用用户
- [ ] 重置用户密码

### 3. 文档审核
**作为** 内容审核员  
**我想要** 审核待发布文档  
**以便** 确保内容质量

**验收标准：**
- [ ] 查看待审核文档列表
- [ ] 预览文档内容
- [ ] 批准或拒绝发布
- [ ] 添加审核意见

### 4. 系统配置
**作为** 系统管理员  
**我想要** 配置系统参数  
**以便** 优化系统功能

**验收标准：**
- [ ] 配置AI参数
- [ ] 配置文件上传限制
- [ ] 配置权限策略
- [ ] 配置通知设置

## 🔐 功能需求

### 1. 仪表板

#### 功能描述
提供系统运行的全局视图，展示关键指标和待办事项。

#### 核心指标

| 指标 | 说明 | 更新频率 |
|------|------|----------|
| 总文档数 | 系统中的文档总数 | 实时 |
| 用户总数 | 注册用户总数 | 实时 |
| 问答准确率 | AI问答准确率 | 每小时 |
| 平均响应时间 | AI问答平均响应时间 | 每小时 |
| 存储使用率 | 文件存储空间使用率 | 每5分钟 |

## 🔗 接口定义

### 1. 获取统计数据

**接口地址：** `GET /api/v1/admin/statistics`

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "totalDocuments": 1234,
    "totalUsers": 5678,
    "totalQuestions": 8901,
    "accuracyRate": 0.89,
    "avgResponseTime": 2.3,
    "storageUsage": 85
  }
}
```

### 2. 获取用户列表

**接口地址：** `GET /api/v1/admin/users`

**请求参数：**
```json
{
  "page": 1,
  "pageSize": 20,
  "role": "USER",
  "status": "ACTIVE",
  "keyword": "张三"
}
```

### 3. 修改用户角色

**接口地址：** `PUT /api/v1/admin/users/{userId}/role`

**请求参数：**
```json
{
  "role": "ADMIN"
}
```

### 4. 获取待审核文档

**接口地址：** `GET /api/v1/admin/documents/pending`

### 5. 审核文档

**接口地址：** `PUT /api/v1/admin/documents/{documentId}/review`

**请求参数：**
```json
{
  "status": "APPROVED",
  "comment": "内容符合规范，同意发布"
}
```

### 6. 更新系统配置

**接口地址** `PUT /api/v1/admin/settings`

**请求参数：**
```json
{
  "enableAI": true,
  "maxFileSize": 50,
  "allowedFileTypes": ["pdf", "doc", "docx"],
  "requireReview": true
}
```
