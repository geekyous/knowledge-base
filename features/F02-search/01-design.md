# F02 - 搜索功能 - 功能设计文档

## 📋 功能概述

搜索功能是企业知识库问答系统的核心功能之一，为用户提供快速、准确的文档和内容检索服务，支持全文搜索、语义搜索和智能推荐。

## 🎯 用户故事

### 1. 智能搜索
**作为** 知识库用户  
**我想要** 通过关键词搜索文档  
**以便** 快速找到所需信息

**验收标准：**
- [ ] 支持关键词输入搜索
- [ ] 显示搜索结果列表
- [ ] 高亮匹配关键词
- [ ] 显示搜索结果相关度
- [ ] 支持搜索结果排序

### 2. 搜索建议
**作为** 知识库用户  
**我想要** 获得搜索建议  
**以便** 发现相关内容

**验收标准：**
- [ ] 输入时显示搜索建议
- [ ] 显示热门搜索词
- [ ] 显示搜索历史
- [ ] 支持点击建议直接搜索

### 3. 分类筛选
**作为** 知识库用户  
**我想要** 按分类筛选搜索结果  
**以便** 精确定位所需内容

**验收标准：**
- [ ] 显示分类筛选器
- [ ] 支持多分类选择

### 4. 高级搜索
**作为** 知识库用户  
**我想要** 使用高级搜索功能  
**以便** 进行更精确的搜索

**验收标准：**
- [ ] 支持时间范围筛选

## 🔐 功能需求

### 1. 搜索主页

#### 功能描述
提供友好的搜索入口页面，包含搜索框、快捷分类、热门搜索等内容，引导用户快速找到所需信息。

#### 页面元素
- **搜索框**：大尺寸、居中显示、支持实时建议
- **快捷分类**：图标+名称+文档数量
- **热门搜索**：搜索词标签，点击直接搜索
- **最近搜索**：显示最近搜索历史

#### 交互行为
- 输入时显示搜索建议（防抖300ms）
- 回车键执行搜索
- 点击搜索建议直接搜索
- 点击分类进入分类文档列表

### 2. 搜索结果页

#### 功能描述
显示搜索结果列表，包含文档标题、摘要、关键词高亮、匹配度等信息，支持分页和二次筛选。

#### 结果展示
```
┌────────────────────────────────────────────┐
│ 搜索框 + 筛选器                              │
├────────────────────────────────────────────┤
│ 搜索结果 (23个结果，用时0.5秒)           │
│                                              │
│ ┌──────────────────────────────────────┐  │
│ │ [文档图标] 产品发布标准流程 v2.0    │  │
│ │ 产品管理 | 2025-12-15 | 1,234 浏览  │  │
│ │ 标签：流程、发布、标准                │  │
│ │ 匹配度：98% [查看详情]              │  │
│ └──────────────────────────────────────┘  │
└────────────────────────────────────────────┘
```

#### 排序选项
- **相关度**：默认排序
- **时间**：最新发布
- **浏览量**：最多浏览
- **点赞数**：最多点赞

### 3. 搜索建议

#### 功能描述
在用户输入搜索关键词时，实时提供搜索建议，帮助用户快速找到相关内容。

#### 建议类型
- **热门搜索**：基于全局搜索统计
- **历史搜索**：用户最近搜索记录
- **相关建议**：基于输入内容的智能建议
- **自动补全**：基于已有内容的补全

### 4. 高级筛选

#### 筛选维度
| 维度 | 说明 | 示例值 |
|------|------|--------|
| 文档类型 | 文档格式类型 | PDF、Word、PPT、视频 |
| 分类 | 文档所属分类 | 人事制度、技术文档 |
| 标签 | 文档标签 | 流程、发布 |
| 时间范围 | 发布时间 | 最近7天、最近30天 |
| 作者 | 文档作者 | 张三、李四 |
| 状态 | 文档状态 | 已发布、草稿 |

## 📊 数据流程

### 1. 搜索流程

```
用户输入 → 前端验证 → 构建搜索请求 → 后端搜索处理 → 结果返回 → 前端展示
                ↓
         [搜索建议]
```

### 2. 搜索建议流程

```
用户输入 → 防抖处理 → 调用建议API → 显示建议列表 → 用户选择 → 执行搜索
```

### 3. 搜索索引流程

```
文档创建 → 内容提取 → 分词处理 → 建立索引 → Elasticsearch存储
                ↓
         [向量化 → Qdrant存储]
```

## 🔗 接口定义

### 1. 基础搜索

**接口地址：** `GET /api/v1/search`

**请求参数：**
```json
{
  "q": "搜索关键词",
  "page": 1,
  "pageSize": 20,
  "categoryId": 1,
  "type": "document",
  "sort": "relevance"
}
```

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "items": [
      {
        "id": 1,
        "type": "document",
        "title": "产品发布标准流程 v2.0",
        "summary": "产品<b>发布</b>流程描述...",
        "highlight": "产品<b>发布</b>标准流程",
        "score": 0.95,
        "category": "产品管理",
        "author": "张三",
        "tags": ["流程", "发布"],
        "createdAt": "2025-12-15T10:30:00Z"
      }
    ],
    "total": 156,
    "took": 23,
    "page": 1,
    "pageSize": 20,
    "totalPages": 8
  }
}
```

### 2. 搜索建议

**接口地址：** `GET /api/v1/search/suggest`

**请求参数：**
```json
{
  "q": "搜索词前缀",
  "limit": 10
}
```

**响应示例：**
```json
{
  "code": 200,
  "data": [
    "产品发布流程",
    "产品发布标准",
    "产品发布时间安排"
  ]
}
```

### 3. 热门搜索

**接口地址：** `GET /api/v1/search/hot`

**响应示例：**
```json
{
  "code": 200,
  "data": [
    "产品发布流程",
    "年假申请",
    "报销政策",
    "新人入职指南"
  ]
}
```

### 4. 搜索历史

**接口地址：** `GET /api/v1/search/history`

**响应示例：**
```json
{
  "code": 200,
  "data": [
    "年假申请",
    "产品发布",
    "技术文档"
  ]
}
```

## 📈 性能要求

### 1. 响应时间
- **简单搜索**：< 200ms
- **复杂搜索**：< 500ms
- **搜索建议**：< 100ms
- **索引更新**：< 5秒

### 2. 并发处理
- 支持100个并发搜索请求
- 搜索请求队列管理
- 超时时间：30秒

### 3. 准确性要求
- **全文搜索准确率**：> 90%
- **语义搜索准确率**：> 85%
- **搜索结果相关性**：> 80%

## 🎨 UI设计规范

### 1. 搜索框设计

#### 主页搜索框
- **尺寸**：宽600px，高60px
- **圆角**：30px（胶囊形）
- **聚焦效果**：蓝色边框+阴影
- **图标位置**：右侧搜索图标

#### 页面内搜索框
- **尺寸**：宽400px，高48px
- **圆角**：24px
- **位置**：页面顶部

### 2. 搜索结果卡片

#### 卡片布局
```
┌──────────────────────────────────────────────┐
│ [图标] 标题                        [匹配度] │
│        摘要（50字以内）                       │
│        分类 | 时间 | 浏览量                 │
│        标签1 标签2 标签3                      │
└──────────────────────────────────────────────┘
```

#### 状态标识
- **文档类型图标**：不同格式不同颜色
